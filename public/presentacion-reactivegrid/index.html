Monitoring ≠ Realtime Analytics


Monitoring

  • Only numeric time series data
  • For storing numbers that change over time (like RRD)
  • Higher resolution of recent data.
  • Lower resolutions for historical data.
  • Frequently plotted via line charts
  • Prediction and forecasting
  • Alarms

Monitoring

Typical queries:


  • Get the latest point from the time series
  • Get the count of events in 5 minute periods for the last 4 hours
  • Get the median in 1 hour increments for the last day

Monitoring

Best example: Graphite

Analytics

  • Data can be anything, arbitrary format
  • Levels of measurement: nominal, ordinal, interval, and ratio
  • Process data in complex ways:
    • Parsing, serialization/deserialization, grouping, aggregators, transformations, custom functions, statistical functions, etc
  • Custom visualization
Sometimes you retrieve what you stored.
Oftentimes you do transformations, aggregations, etc.
Queries as pure functions that take all data as input is the most general formulation.

Query = Function(All Data)

              
function pageViewsOverTime(allData, url, start, end) {
    count = 0;
    for(data: allData) {
        if(data.url == url && data.timestamp >= start && data.timestamp <= end) {
            count++;
        }
    }
    return count;
}

Analytics

You should be able to run ad-hoc queries against all of your data to get results, but doing so is unreasonably expensive in terms of resources.
              
SELECT
    P.Brand,
    S.Country AS Countries,
    SUM(F.Units_Sold)
FROM Fact_Sales F
INNER JOIN Dim_Date D    ON F.Date_Id = D.Id
INNER JOIN Dim_Store S   ON F.Store_Id = S.Id
INNER JOIN Dim_Product P ON F.Product_Id = P.Id
WHERE
    D.YEAR = 1997
AND P.Product_Category = 'tv'
GROUP BY
    P.Brand,
    S.Country
